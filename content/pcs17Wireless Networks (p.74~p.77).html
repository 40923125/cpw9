pcs17Wireless Networks (p.74~p.77)

Overview

In this module, we will discuss wireless networks. We will focus on wireless network security and how to access a network via Wi-Fi.

Wireless Security

Wireless security is extremely important when setting up a wireless network. Without security, anyone can gain access to files and folders, see all wireless transmissions, introduce their own malware, use your wireless connection to access illegal content or launch an attack through your ISP, or download harmful content.

Wireless networks use the air as their transport medium by using the radio frequencies. This means that the medium is not secure because any user within the distance limitation can access the wireless network, if encryption and passwords are not utilized.

There are four easy steps to secure an access point:

1. Lock down the router or access point.

2. Configure the service set identifier (SSID), and disable SSID broadcasts.

3. Configure wireless security.

4. Limit users by media access control (MAC) address.

Lock Down Access Point and Access

There are several methods for locking down the access point. The more methods you use, the more secure the network will be. You can lock down the access point or router by changing the default password to a strong password. You can also lock down the system by disabling wireless web access so only a directly connected computer can manage the device. You can disable remote management so the device cannot be accessed over the Internet. You can only allow access through the more secure Hypertext Transfer Protocol Secure (HTTPS). Finally, you can disable universal plug-and-play.

Additionally, you should change the network access by changing the default IP address for the network. It’s also good practice to limit the number of addresses available through the dynamic host configuration protocol (DHCP). You can change the default SSID, and disable its broadcast so users must know what it is before connecting to it.

SSID — Service Set Identifier

One method of securing your wireless network is to disable the broadcast of the SSID. The SSID, or service set identifier, is essentially the name of the wireless network. To secure the wireless network, you should first change the SSID to something other than the default. If a user sees a default SSID, it can often be assumed that there is no administrative password in place on the wireless access point; therefore, the access point becomes the target of attacks. Also, if there are multiple access points in the same area, and they all have the default name, it becomes difficult to determine which access point belongs to which network. When changing the default SSID, do not use your family’s name, company name, or location, as this makes your access point a source of information to use against you and target your network. Once all your wireless clients are configured, it is a good idea to disable the SSID broadcast. This will prevent the access point from announcing its presence in the area. Once the SSID is disabled from broadcasting, users will no longer find the wireless network shown in their available networks, unless they have connected to it previously. This means that you have to set your client machines to connect to the network even when it is not broadcasting.

Wi-Fi Protected Access

Due to the nature of Wi-Fi being transmitted through radio frequencies, the data transferred is susceptible to being viewed and captured by any other radio receiver. Encryption and network security have become a paramount concern for IT professionals at all levels. If a Wi-Fi network is set up without encryption, it is “open,” and anybody with packet sniffing software in the vicinity can capture the transmissions between the host and the wireless access point. Public cafes and retailers offering open access do so as a courtesy, but users must be aware that any online login information, such as connecting to a bank, Facebook, or Twitter, is not secure and is susceptible to capture. prospective technicians must know the current encryption types and best practices to secure a WLAN.

Encryption Types

In order to provide security for wireless networks, the 802.11 standards provide for three major types of password protection/encryption: wired equivalent privacy (WEP), Wi-Fi protected access (WPA), and Wi-Fi protected access 2 (WPA2).

Wired Equivalent Privacy (WEP)

In the original 802.11 standard, the only security provided was WEP, which at the time claimed to be as secure as a wired network connection. This wireless connection was secured using encryption with a 40-bit WEP preshared key (PSK) that all access points and client devices would use. This symmetric encryption was a good idea, but since the key was so small (40 bits), it was quickly broken by attackers.

To solve this, WEP began using a 64-bit key, and could support keys up to 128 bits, which would provide stronger encryption. Unfortunately, the WEP algorithm used for encryption also relied on using an initialization vector (IV) in combination with the PSK to create the encryption, and the IV is created using only 24 bits, which means there are only 16,777,216 possible values. This limited combination value means that the IV will begin repeating eventually (in five to seven hours) and an attacker capturing packets can get two with the same IV. This is known as a keystream or IV attack. Many open-source utilities exist that can crack WEP passwords within just a few minutes on modern hardware. For this reason, WEP is insecure and should never be used in your networks.

Wi-Fi Protected Access (WPA)

The Institute for Electrical and Electronics Engineers (IEEE), in conjunction with the Wi-Fi Alliance, began developing a solution to WEP’s shortcomings. In 2003, the Wi-Fi Alliance released the WPA standard as a stopgap to replace the now insecure WEP. WPA is built around the same Rivest Cipher 4 (RC4) stream cipher but used a 48-bit IV in place of WEP’s 24-bit IV. This upgraded IV uses a temporal key integrity protocol (TKIP). TKIP is an IEEE standard designed to provide more secure encryption using a 128-bit dynamic per packet encryption key. TKIP is a suite of algorithms designed as a firmware upgrade for WLAN network interface cards (NIC) on the client side, but not necessarily access points manufactured before 2003. TKIP encrypts each data packet with a unique encryption key, and the keys are much stronger than WEP due to the 128-bit length. To increase key strength, TKIP includes four additional algorithms.

In addition to TKIP, the WPA standard uses message integrity check (MIC) to confirm that data was not modified in transit as opposed to the cyclic redundancy check (CRC) used in WEP. MIC uses a “hash” to verify that a message has not been altered. WPA supports both PSK mode and enterprise mode. In enterprise mode, users are required to authenticate before exchanging their keys and each key between the client and access point is only temporary, reducing the chance of compromise. Unfortunately, WPA also suffers security flaws centered on the use of passphrases and PSK key management, and now is also considered to be insecure.

Wi-Fi Protected Access 2 (WPA2)

Wi-Fi protected access version 2 (WPA2) replaces both WEP and WPA because of the insecurities with the 24-bit and 48-bit initialization vectors. IEEE ratified WPA2 as 802.11i in 2004. WPA2 requires stronger encryption and integrity checking, using counter mode cipher block chaining message authentication code protocol (CCMP) . While WPA2 supports a preshared key and the earlier 802.1x standards, it specifies advanced encryption standard (AES), which is the most secure. AES can use a 128, 192, or 256-bit key, and is the same algorithm used worldwide as well as by the United States government to protect classified networks.

Using WPA2 creates a secure wireless network, but only when used with good passwords and shared keys. WPA2 also supports enterprise mode for centralized user authentication (used in corporate networks) and a personal mode for PSK support (used in most homes and small offices). The key to security with WPA2 is a strong password, since the current method of breaking into a WPA2 network is to use brute force attacksagainst the PSK. The minimum password is eight characters and the maximum is 63 characters. The ideal password length is 12–16 characters following strong password protocol of not using standard dictionary words and using a mix of uppercase and lowercase letters, numbers, and symbols. If your key is long and complex, the attackers will not be able to penetrate your WPA2 wireless networks. For more information about securing WPA2, read this article.

Wi-Fi Protected Access 2 Enterprise

WPA2 enterprise is a client/server model using the remote authentication dial-in user service (RADIUS) protocol found in larger corporate networks. The RADIUS server runs in the application layer using UDP (user datagram protocol) as a transport method, and is usually a software background service running on a UNIX or Windows server. WPA2 enterprise eliminates a shared key and replaces it with a username and a password to connect to the corporate Wi-Fi network.

Media Access Control (MAC) Filtering

MAC filtering is another method of securing a network (both wired and wireless). MAC filtering relies on the layer 2 address of the network interface card (NIC). A MAC address is a 48-bit address individually assigned to each network interface card by the manufacturer, with the first 24 bits identifying the manufacturer and the last 24 bits identifying the specific individual network interface card. MAC filtering is configured at the wireless access point or wireless router by creating a whitelist (allowed) or blacklist (denied) of specific MAC addresses. It is best to let all devices connect to the access point first so their MAC addresses will appear automatically in the list, and they don't have to be entered manually. Note that people are not identified by the MAC address, instead it is their devices that are identified, so it is important that the devices remain secure through additional means. MAC filtering is part of a total security solution, not a solution in itself.

Additional Wireless Security Methods

Wireless security can also be enhanced by choosing static versus dynamic IP addressing. By utilizing a static address, each client must manually be configured for the subnet, gateway, and DNS server, making this a more secure option for a wireless network. Conversely, a dynamic scheme could be utilized, where addresses are automatically assigned using a DHCP server. While this eases setup through the use of automatic configuration, if a malicious user connects to the network, they are also automatically configured if other steps — such as MAC filtering, use of WPA2 AES passwords, and changing default settings — are not taken.

In a small office/home office (SOHO) network, manually configuring IP addresses might be of value, provided you have a fixed number of machines. However, with the ubiquity of cellular devices and other small personal computing devices, accessing a network is difficult to manage and much less practical than using DHCP. When using DHCP in your networks, you can minimize the number of DHCP addresses available to prevent unwanted use of your network by malicious users. Further, most SOHO routers today include a guest network that will allow users access to only services you configure at the router (usually Internet but not LAN access).

Lastly, you need to be mindful of the placement of your access points for effective and secure coverage. If you are using more than one access point, you must plan to prevent the access points from causing interference with each other. This is done by separating them logically by using distant channels to prevent overlap. Also, you should ensure that the power levels of your access point radios are set low enough to keep their signals within your building, but high enough to provide full coverage throughout your area. If your signal is too strong, a malicious user could hijack wireless signals from the parking lot or across the street. This is known as “wardriving.”

Accessing a Network Via WiFi

Accessing a wireless network from a Windows computer is fairly straightforward, but as with many other features we’ve discussed, the process is similar in Windows Vista and Windows 7, but differs a bit in Windows 8.1 and Windows 10.

To access a wireless network, you’ll need two pieces of information:

  • The name of the network (SSID).
  • The password for the network (security key).

Windows Vista and Windows 7

From Windows Vista or Windows 7, the easiest way to access the network is by using the Wi-Fi icon in the system tray on the lower right of your Desktop.

Screenshot of Windows Taskbar showing the Wi-Fi icon in the system tray.

Clicking on the Wi-Fi icon brings up a list of available wireless networks, along with a status indicator showing the strength of the signal for each network. Click on the name of the network to which you would like to connect. If you don’t see your desired network, make sure that you’re close enough to the router for your computer to receive the wireless signal.

List of available networks.

Finally, enter the password or security key for the network.

Entering the wireless network password.

Windows 8.1

In Windows 8.1, using the Wi-Fi icon will work if you are in the Windows Desktop, just like in Windows Vista or Windows 7. There is also another way to access wireless networks using the Metro interface.

First, open the Charms menu on the right of the screen (or use the keyboard command Windows key () + C) and click on the Settings icon.

Next, click on the Wi-Fi icon that appears in the Settings menu.

Tips for Accessing Wireless Networks

If you’re trying to connect to a wireless network from a laptop computer, you should ensure that the wireless card is enabled. Some computers allow the user to turn off the wireless card when not in use to conserve battery power. Toggling the wireless card on and off in these cases is typically achieved using a function (Fn) key shortcut. See the manufacturer’s instructions for details pertaining to your specific laptop.

Increasingly, free wireless access is offered by airports, cafes, and retailers as a courtesy. These are open, unsecured networks. Remember that any online login information, such as connecting to a bank, email, Facebook, or Twitter, is susceptible to capture by anyone nearby. Thus, care should be taken when using a free public wireless network that you do not log into your sites unless you have added a virtual private network (VPN) through a third-party Internet security suite or from your antivirus program.

For networks that you connect to frequently, you can click the "Connect automatically" box so that your computer will automatically connect to the network whenever you are in range. This will prevent you from having to choose the network and enter the password every time you want to connect.
